V3.4 — Triple Resolution, All Fixes Applied

V3.4 triples the resolution and fixes all remaining issues:

  • Triple resolution: Grid 800→1384 (640K→1.9M vertices per model). All 4 models now 40-64 MB.
  • No spikes on ANY model: Higher resolution + 5×5 morphological smoothing on boundary + 20 Taubin + 3 Laplacian = smooth edges everywhere
  • No gold border on intelligent models: Border vertices now use the nearest crest color (sampled via KD-tree), not artificial gold. The back is the same color as the front everywhere.
  • Split parts same thinness as full crest: All models use identical RELIEF_DEPTH (0.6) and BACK_DEPTH (0.1) — no model is thicker than another.
  • Same smoothing on all 4 models: Model 1 was perfect, so the same parameters (20 Taubin + 3 Laplacian) are applied to all.

Model Sizes (All Tripled)

  • Full Crest (No White): 63 MB (was 20 MB) — ~1.6M vertices
  • Full Crest (Intelligent): 64 MB (was 21 MB) — ~1.65M vertices
  • Split Parts (No White): 41 MB (was 13 MB) — ~1M vertices
  • Split Parts (Intelligent): 56 MB (was 17 MB) — ~1.45M vertices

Technical Specs

  • Grid: 1384×1384 = 1,915,456 candidate vertices (tripled from 800×800)
  • All 4 models: 40-64 MB each
  • Source: 4096×4096 (4K, transparent BG)
  • Relief depth: 0.6 units (front), back thickness: 0.1 units (same for all)
  • Height map blur: 31×31 Gaussian
  • Alpha blur: 15×15 Gaussian
  • Boundary smoothing: 5×5 morphological close+open
  • Mesh smoothing: Taubin 20 iter + Laplacian 3 iter (all models)
  • Intelligent border: Nearest-crest color via KD-tree (NOT gold)
  • Watertight (manifold) — 3D-printable
